Export a file

Many kinds of files besides the original form can be exported with menu [File] [Save As] and can be rendered in other softwares. But as some of them are unfinished or do not meet the requirements of the format, some of information are generally lost. In order not to be confused later to notice that information cannot be transferred smoothly, I recommend you to check how it works beforehand.

LWO (LightWave3D) (only shareware version)

It is available with LWO2 form supported in LightWave3D 6.0 and later and LWOB form also supported in LightWave3D 5.x.

Visible objects only
Only visible objects are exported. Invisible objects are ignored.
Not freeze patches
When you want to use not the subdivision method of Metasequoia but SubPatch (MetaNURBS in LightWave3D 5.x), select 'ON' and apply subpatch in LightWave3D side.
LWO2
It is the form supported in LightWave3D 6.0 and later. This form can contain the UV coordinates, but a part of UV coordinates will be lost without LightWave3D 6.5 and later. It can also be read in LightWave3D 5.x. with 'OFF' on this check, but plural objects can be composed to one object and the number of vertices and that of faces are limited to 65535.
Absolute file path
If you select 'ON' in this check, the file name of texture image is expanded to absolute path. If you check 'OFF', the file name assigned to the material is saved without change. (If originally selected as absolute path, it is saved without change.)

3DS (3D Studio) (only shareware version)

It automatically changes to a triangle.(*1)

Export is discontinued in case the number of vertices or that of faces exceeds 65535.

When the object name exceeds 9 letters, the material name 15 letters, and the texture file name 12 letters (including an extension), it cannot normally be read in other softwares.
This format is used in not 3ds Max but 3D Studio for MS-DOS. The other format is recommended to transfer data to 3ds Max because this format is very old, has many limitation and is less compatible with 3ds Max.

X (Direct3D)

It is saved as the text form (ASCII) introduced in DirectX 2.

The plural objects are composed to one object.

It saves the file name only without pass (file name) in texture.(*1)

UV coordinates and vertex colors are assigned to each vertex in a .X file. If different UV coordinates or vertex colors are assigned to each vertex sharing the same face, the number of vertices to be exported will increase when you select 'ON' in the check of [UV mapping] or [Vertex color].

Visible objects only
Only visible objects are exported. Invisible objects are ignored.
Normal vector
It selects whether to export normal vectors. If you reset in reading side of the program, the file size lessens when you unselect this option.
UV mapping
It selects whether to export UV mapping necessary for texture. If you don't use texture or reset mapping in the receiver side of the software, the file size lessens when you unselect this option.
Vertex color
It selects whether to export the vertex color. If you don't use the vertex color, the file size lessens when you unselect this option.

OBJ (Wavefront) (only shareware version)

The plural objects are composed to one object.

Material is saved as material library in the same extention.mtl file. Please notice the following points concerning texture when you export material library.

  • It is necessary to put texture file in the same folder with OBJ file in the receiver side of the software because only the file name of texture without pass (file name) is exported.
  • The file name of texture must not contain blanks.
  • The receiver side of the software cannot sometimes read the file when the file name of texture includes Japanese.
  • Please export after confirming which file forms can be read as texture in the receiver side of the software.
Visible objects only
Only visible objects are exported. Invisible objects are ignored.
Grouping
Each object can be picked out as an independent group when the receiver side of the software is available. But, the receiver side of the software cannot sometimes read the file when the file name of an object includes multi-bytes language (like Japanese).
Normal vector
It selects whether to export smooth information. In case you reset in the receiver side of the software, the file size lessens when you unselect this option.
UV mapping
It selects whether to export UV mapping necessary for texture. If you don't use texture or reset mapping in the receiver side of the software, the file size lessens when you unselect this option.
Invert V
As how to arrange values of V(vertical) direction in UV mapping becomes upside down, it becomes necessary to choose the applicable one.
Material library
It exports material library.mtl file containing material and texture information.
Freeze patches
Please select "OFF" in this option when you use Subdivision Surface in the receiver side of the software.
Return code
It cannot sometimes be read normally when you do not select the new line code of OS available for the receiver side of the software.
