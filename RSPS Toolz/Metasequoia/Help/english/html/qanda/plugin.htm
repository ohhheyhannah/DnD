FAQ - Plug-in

Q1. What is plug-in?

A1. Extension function
It is a function which can add afterwards. In Metasequoia, a plug-in is provided as a DLL file. Some default functions like [Create text] are implemented as plug-ins.
As Developing kit(SDK) is open to the public, you can make your own plug-ins if you have programming knowledge on C++ and Windows.

Q2. Where can I get plug-ins?

A2. See the home page of a web site
Some links to plug-ins which users made are connected in the official homepage. Many other plug-ins are also open to the public.

Q3. How can I develop plug-in?

A3. Get a plug-in SDK
You can download the plug-in SDK from the official homepage.
A C++ development software is also necessary for developing plug-in. Visual C++, Visual Studio and C++Builder are confirmed to be available.

Q4. I cannot use plug-in.

A4-1. Serial ID has not registered.
Use of plug-in is limited only to import files if serial ID has not registered. It is necessary for purchasing the license to use plug-ins.
A4-2. It has not installed correctly.
See the document attached to the plug-in if it exists when you want to know how to install the plug-in. If there is not any specific instructions, press the [install] button on the screen shown by menu [Help] [Plug-in] and select the plug-in DLL.
