Q1. I paste pattern mapping (texture), but it changes to a strange color without showing pattern.

A1. UV value has not assigned.
Assign UV value in [UVedit] command or select other formats except UV in [Mapping] command or Material Property.

Q2. I select alpha mapping in Material Property, but it does not work on 3D screen.

A2-1. Video memory is insufficient.
As video memory (VRAM) is limited in Direct3D or OpenGL mode, it cannot be shown when available capacity becomes insufficient.
You can take measures like:
  • Make the size of the picture for texture smaller.
  • Reduce the resolution or the number of colors of the picture.
  • Change to the video card with more video memory.
  • Reduce the value in [Max resolution of texture] in [Preview] page of menu [File] [Configuration].
But the setting itself of mapping image is effective even if it is not shown, so you do not have to worry about that according to the situation.
A2-2. Image file for texture is not put in the appropriate folder.
You need to put image folder in the folder selected in [Folder of mapping bitmaps] in menu [File] [Configuration].
A2-3. View Property is not checked.
Please check in menu [View] [Show precisely] and [Show textures].

Q3. I select bump mapping in Material Property, but it does not work on 3D screen.

A3. It's a specification
Bump mapping cannot be shown in any view mode in this version. It is not reflected on the result in [Rendering]. It will be possible to be supported in future.

Q4. I cannot undo the opration of editting mapping images by using [Paint] command

A4. Call in the paint panel
Please call the menu [Edit] [Undo] in the paint panel.
