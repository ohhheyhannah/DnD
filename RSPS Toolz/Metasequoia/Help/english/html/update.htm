Release note

What's new in Ver3.1.5

  • Improved
    • Anguwrap can unwrap correctly for an image with different width and height.
  • Bug fixes
    • A zoom for an orthographic view is changed after changing the selection when [Selected part] is selected for [Center of rotation] in the configuration.
    • A right/left is reversed for STL exporter.
    • 'Anguwrap' occurres an error in the specific condition.
    • 'power' for a material cannot be set in Python script.

What's new in Ver3.1.4

  • Changed
    • [Simple] button is always off at the startup time whether it was on or off at the last running time.
  • Bug fixes
    • A preview for a bevel will be displayd incorrectly when the [Extrude faces] menu is called after the [Extrude] command.
    • An application crashes when the Anguwrap plug-in is called without selections of any lines.

What's new in Ver3.1.3

  • Bug fixes
    • An unique ID of the object is broken after canceling the [Round] menu.
    • [Edge crease for Catmull-clark] sometimes does not change anything when an edge in a triangle is selected.
    • A mirroring object is not updated in the view after switching [Smoothing shading] by the right-click menu in the object panel.
    • A painting over a border of a non-square image in the paint panel sometimes makes an invalid stroke.
    • A face and a vertex on a mouse cursor is not highlighted when [Apex] is selected in [Vertex color] command.
    • Selected faces in a patch of Spline Type 1/2 are not displayed correctly in a Direct3D display when [Change the colors of lines in selected faces] is off.
    • OpenGL does not display double-sided faces correctly.
    • An edge crease is not loaded correctly in the case of setting a vertex color in loading a .mqo file
    • MQMateria::GetMaterialFromUniqueID() returns an invalid material.

What's new in Ver3.1.2

  • Bug fixes
    • An error sometimes occurs by an undo or a redo after creating or deleting an object.

What's new in Ver3.1.1

  • Improvement
    • .obj files include group names in the utf-8 encoding can be loaded normally.
    • Parameters for multiple directional lights can be get and set with the new plug-in SDK.
    • An operation for a selection in the object/material is compatible with Ver3.0 when "ListMultiSelectionWithAlt=1" is written in "metaseq.ini".
  • Bug fixes
    • UVs are broken after executing the [Mesh smoothing] menu.
    • The selection in an object/material panel is not restored after an undo.
    • A hozitontal/vertical rotation of the view with Shift + right button dragging cannot be performed in the [Select] command.
    • UVs are broken after executing the [Bevel] command.
    • A size of a brush in the [Vertex color] command cannot be changed.
    • Loading a .rsd file sometimes causes an error.
    • The 'lightAttenuation' property for an object cannot be set in the Python script.

What's new in Ver3.1.0

  • System
    • The window position at the time of the last quit will be restored correctly regardless of the position of the task bar.
  • Configuration
    • Added a specification method of a file path for a mapping image. It can be selected from [Auto], [Absolute path] or [Relative path].
    • A file can be associated without an administrator privileges at the time of the running on Windows Vista or later version of Windows.
  • Panel
    • Multiple items can be selected in the object panel and material panel.
    • Selection of vertices, lines or faces in the object panel or material panel has been changed to Alt key, and deselect has been changed to Ctrl+Alt keys.
    • A new object or material will be created just under the current one when the [New] or [Copy] button in the object and material panel is pushed with Ctrl key.
    • Children are moved together when a parent object is duplicated or a hierarchy of the parent object is changed.
    • Added [Merge selected] and [Set as default] on the [Misc] menu in the object panel.
    • Added [Reserve triangles] for the Catmull-Clark patch on the right-click menu in the object panel and in the object property.
    • A parameter for multiple objects can be changed together by selecting multiple objects on the list in the object property.
    • A duplicated object or material inherits a name from the original one.
    • A bump map image or a normal map image can be specified for the [Bump] mapping in the material property.
    • Added [Double sided] in the material property.
    • Pan can be operated by dragging with a space key in the paint panel.
    • Added [Convert to normal map] in the menu of the paint panel.
  • Command
    • A selection is not cleared when you click the place which does not have anything with Shift key.
    • A selection is toggled with Shift key for all operations except Brush in [Select].
    • Lines are selected by [Belt] in [Select] when [Face] in the edit options is off.
    • A new selection will be not over the selected lines by [Edge loop] in [Select].
    • An operation for [Route] in [Select] is changed from a drag to clicks.
    • Selections in the other objects will be cleared when a current object is changed by [CurObj] in [Select] and so on.
    • Lines can be selected by [Route] in [Select].
    • Faces can be selected by [Brush] in [Select].
    • An operation can be performed with opening the numeric dialog in [Rotate].
    • A center position of the handle will be not changed after inputting a rotation angle and pushing OK button in [Rotate].
    • A wire between the mouse position and the vertex is displayed while you create a new face in [Create].
    • A vertex can be placed horizontally or vertically with Shift key in [Create] or [Pull].
    • A vertex can be snapped to the grid in [Pull].
    • UV coordinates will be hardly changed from the original shape by [Bevel] or the bevel mode in [Extrude].
    • [Twist], [Bend], [Shear] and [Weight] support a symmetric editing.
    • A vertex will be snapped to the nearest one when you click a right button while dragging a start point or an end one in [Twist].
    • A preview will be refreshed after modifying an alpha channel when a texture mapping with an alpha channel is edited in [Paint].
    • A creased edge for Catmull-Clark patch can be specified by [Weight].
  • Menu
    • A rendered image can be sent to the clipboard in [Rendering].
    • UV coordinates will be hardly changed to appearance after executing [Mesh smoothing].
    • A coordinate of the base position can be specified in [Align faces].
    • Added [Show subdivision wires] to display wires for a Catmull-Clark patch.
    • Added [Edge crease for Catmull-Clark] to make a creased edge.
    • An efficient unwrapping of UV coordinates can be performed by the 'Anguwrap' plug-in.
  • Preview
    • Patch of spline type 1/2 can be displayed with a smoothing.
    • A high power of a specular does not produce an exposure in OpenGL preview.
  • Export/Import files
    • TIFF images can be imported and exported.
  • Compatibility
    • An opening edge makes the same Catmull-Clark subdivision surface as Ver3.0 when "KeepOpenEdgeVertex=1" is written in "metaseq.ini".
    • [Continuous face], [Belt] and [Continuous line] in [Select] command selects an unselected face additionally at the time of clicking it without pressing Shift-key when "ReserveSelectionWithoutShift=1" is written in "metaseq.ini".

What's new in Ver3.0.5

What's new in Ver3.0.3

    • When you specify a mapping image with a relative path in the Material Panel, the path will be shown as an initial folder.
    • Supported exporting STL files.
  • Bug fixes
    • A route selection in [Select] command sometimes causes an error.
    • [Bevel] command sometimes causes an error.
    • A non-highlight line is sometimes selected in [Knife] or [Wire] commands.
    • A texture in a background image is not interpolated with a Direct3D display.

What's new in Ver3.0.2

  • Improvement
    • When you specify a mapping image file at the same or under folder of the .mqo file, the file name is set as a relative path.
    • The V direction can be changed when you export a .dae file.
  • Bug fixes
    • The file insert dialog is behind the main window when you drop a file from Explore.
    • A mirroring shape is not updated when you change a local coordinate without a shape interlock in [Local] command.
    • An application freezes when you drag an item for a long time in the Object Panel or the Material Panel.
    • A view is not updated when you change a type of a mirroring by a right-button clicked menu in the Object Panel.

What's new in Ver3.0.1

  • Improvement
    • The folder that contains the editing file is set as an initial one when 'Save As' is called.
  • Bug fixes
    • A Catmull-Clark patch causes an error when the same vertex in the one face is shared.

What's new in Ver3.0.0

  • System
    • Improvement of a memory management.
    • A direction of a dolly by a wheel is reversed.
    • A dolly can be operated in the larger range.
  • Configuration
    • Added a font setting.
    • Added a selection of a format for help files.
    • Added a transparency for lines in back side of faces.
    • Added an anti-alias setting for Direct3D.
    • Added view operation settings for Alt + Left/Middle/Right buttons.
    • Added an inverse of an direction of a mouse operation for zoom or dolly.
    • Added a setting for a method of moving non-symmetry vertices.
    • Deprecated settings for text drawing in Direct3D and coordinate transformation in OpenGL.
    • Added a setting for a size of a handle.
  • Panel
    • A non-symmetry vertex is not snapped to the X=0 when the vertex is scaled in the edit panel.
    • Multiple materials can be selected and the properties of them are changed together in the material panel.
    • Supported three parallel lightings.
  • Command
    • Added [Growing], [Route] and [Edge loop] modes in [Select] command.
    • When a handle is moved in [Scale], [Rotate] command, it is snapped to the position based on the center.
    • The center position for scaling or rotating by [Scale] or [Rotate] command can be selected from a center position of a local coordinate, a zero point and a center position of selected parts.
    • A handle can be rotated correctly when the parent object is scaled in the [Local] command.
    • Separation to the left or right by mirroring becomes less likely to occur by rotating in [Local] command.
    • A front side of face is always created when the new face are pulled from a line in [Pull] and [Wire - Create a triangle] command.
    • A new triangle can be created by dragging between the edges of the two lines that share the same vertex in [Wire - Create a triangle] command.
    • The numeric display in the property of [Local] command is updated when the hierarchy is changed in the object panel.
    • Vertices are not separated  at the same position, and a hole is not made by [Bevel] command.
    • Supported to twist by 15 degrees by dragging with Shift key in [Twist] command.
    • Added [Magnet], [Brush], [Expand], [Rect], [Rope], [Align U] and [Align V] modes in [UV edit] command.
    • Supported symmetry editing in [Move], [Scale] and [Rotate] mode in [UV edit] command.
    • UV vertices are shift along U or V direction only with dragging the Shift key by [Move] and [Magnet] mode in [UV edit] command.
    • UV vertices are rotated or scaled along the dragged vertex by [Rotate] and [Scale] mode in [UV edit] command.
    • Supported rotation of the initial lattice in [Lattice] command.
    • Supported aligning the direction along the local coordinate in [Lattice] command.
    • Added a brush mode in [Vertex color] command.
    • Added [Sculpt] command.
  • Menu
    • FOV is not changed when a [Fit to view] menu is called.
  • Preview
    • Improvement the displaying of flat, cylinder and sphere mapping on the curved surface by patch type 1/2.
    • Support Direct3D displaying using DirectX 9.0.
    • Support displaying textures in three orthogonal views.
    • Back side of lines are thinly displayed.
  • Export/Import files
    • Fixed loading 8-bit .tga files with alpha palette.
    • Supported exporting .dae(COLLADA 1.4) files.
    • When a file except .MQO is loaded, textures are loaded with the relative path based on a folder contains the file.
    • Files supported by Import plug-ins are displayed when [All supported files] is selected in the file import dialog.
    • Supported PSD files loading for mapping images and background images. (Read only)
